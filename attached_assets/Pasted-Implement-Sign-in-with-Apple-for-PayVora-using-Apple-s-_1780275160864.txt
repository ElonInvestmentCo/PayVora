Implement Sign in with Apple for PayVora using Apple’s official AuthenticationServices framework and Expo’s native expo-apple-authentication package.

Requirements:

1. Remove any custom Apple login buttons, icon-only buttons, or redesigned Apple branding.
2. Use ONLY Apple’s native authentication UI:
    * AppleAuthenticationButton (Expo)
    * ASAuthorizationAppleIDButton (native iOS equivalent)
3. Follow Apple’s Human Interface Guidelines exactly.
4. The Apple Sign In button must appear above or before alternative login methods.
5. Do not modify Apple’s colors, logo, typography, or button styling.

Authentication Flow:

* User taps native Apple Sign In button.
* Call AppleAuthentication.signInAsync().
* Request:
    * FULL_NAME
    * EMAIL
* Handle:
    * Successful authentication
    * User cancellation
    * Authentication failures
    * Network failures
* Retrieve:
    * user identifier
    * email (when provided)
    * full name (when provided)
    * identityToken
    * authorizationCode

Backend Flow:

* Send identityToken and authorizationCode to the backend.
* Verify credentials with Apple’s servers.
* Automatically create a new account if one does not exist.
* Automatically sign in existing users.
* Use a single unified login/signup flow.

Secure Storage:

* Do NOT store Apple credentials locally.
* Store only the application’s authenticated session token.
* Use expo-secure-store for session persistence.

Platform Requirements:

* Render Apple Sign In only on iOS.
* Use Platform.OS === ‘ios’.
* Hide Apple Sign In completely on Android and Web.
* Prevent unsupported platform imports from crashing the application.

Expo Configuration:

app.json

{
“expo”: {
“ios”: {
“usesAppleSignIn”: true
},
“plugins”: [
“expo-apple-authentication”
]
}
}
